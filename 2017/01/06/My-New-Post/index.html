Mac安装MySQL 5.7.9版本 出现sql_mode=only_full_group_by问题

2017-01-06

解决办法：
MySQL版本：5.7.16

OS版本：OS X EI Capitan（ 10.11.5 ）

设置编码时需要修改/etc/my.cnf的内容，但是在/etc/下并没有my.cnf，
但是，在/usr/local/mysql/support-files下有一个my-default.cnf，只要把这个文件复制到/etc/下，再改名就好了。

另外，如何解决Mac Mysql 乱码问题：

顺便说一下修改的步骤：

  1. 我修改的并不是utf8，而是utf8mb4，区别请自行查询

  2. 先登录到mysql：
    mysql -u 数据库帐号 -p

  3. 查看默认安装的字符集：
    mysql> show variables like ‘%char%’;（分号属于命令行）

  4. 使用775权限：
    sudo chmod 775 /etc/my.cnf

  5. 使用vim打开my.cnf，编辑：
    vi /etc/my.cnf

  6. 在[mysqld]上边添加

    default-character-set=utf8

在[mysqld]下边添加：
character-set-server=utf8

键入:wq保存并退出，再次进入查看，看到

character_set_database   | utf8

就是修改成功了
