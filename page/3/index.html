Git 本地分支管理

查看、切换、创建和删除分支

git br -r # 查看远程分支

git br # 创建新的分支

git br -v # 查看各个分支最后提交信息

git br –merged # 查看已经被合并到当前分支的分支

git br –no-merged # 查看尚未被合并到当前分支的分支

git co # 切换到某个分支

git co -b # 创建新的分支，并且切换过去

git co -b # 基于branch创建新的new_branch

git co $id # 把某次历史提交记录checkout出来，但无分支信息，切换到其他分支会自动删除

git co $id -b # 把某次历史提交记录checkout出来，创建成一个分支

git br -d # 删除某个分支

git br -D # 强制删除某个分支 (未被合并的分支被删除的时候需要强制)

分支合并和rebase

git merge # 将branch分支合并到当前分支

git merge origin/master –no-ff # 不要Fast-Foward合并，这样可以生成merge提交

git rebase master # 将master rebase到branch，相当于： git co && git rebase master && git co master && git merge

git apply –check ../sync.patch #测试补丁能否成功

git pull # 抓取远程仓库所有分支更新并合并到本地

git push origin # 创建远程分支， origin是远程仓库名

git push origin : # 创建远程分支

git push origin : #先删除本地分支(git br -d )，然后再push删除远程分支

Git远程仓库管理

GitHub

git remote show origin # 查看远程服务器仓库状态

创建远程仓库

Mac安装MySQL 5.7.9版本 出现sql_mode=only_full_group_by问题

发表于 2017-01-06

解决办法：
MySQL版本：5.7.16

OS版本：OS X EI Capitan（ 10.11.5 ）

设置编码时需要修改/etc/my.cnf的内容，但是在/etc/下并没有my.cnf，
但是，在/usr/local/mysql/support-files下有一个my-default.cnf，只要把这个文件复制到/etc/下，再改名就好了。

另外，如何解决Mac Mysql 乱码问题：

顺便说一下修改的步骤：

  1. 我修改的并不是utf8，而是utf8mb4，区别请自行查询

  2. 先登录到mysql：
    mysql -u 数据库帐号 -p

  3. 查看默认安装的字符集：
    mysql> show variables like ‘%char%’;（分号属于命令行）

  4. 使用775权限：
    sudo chmod 775 /etc/my.cnf

  5. 使用vim打开my.cnf，编辑：
    vi /etc/my.cnf

  6. 在[mysqld]上边添加

    default-character-set=utf8

在[mysqld]下边添加：
character-set-server=utf8

键入:wq保存并退出，再次进入查看，看到

character_set_database   | utf8

就是修改成功了
